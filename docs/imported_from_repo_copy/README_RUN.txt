### كيفية تشغيل نظام AGL

1. تأكد أن بيئة Python الافتراضية موجودة في `D:\AGL\.venv` وتعمل (Ollama يجب أن يكون قيد التشغيل إن كنت تستخدمه كموديل محلي).
2. افتح المجلد `D:\AGL\repo-copy` في مستكشف الملفات.
3. اضغط مرتين على `Start_AGL.bat`.
   - الملف سيقوم بتفعيل البيئة الافتراضية، فتح المتصفح على http://127.0.0.1:8000، ثم تشغيل الخادم (server.py).
4. عند ظهور واجهة المتصفح، ابدأ بإرسال الأسئلة.

ملاحظات واشتراطات:
- لا تغلق نافذة الدوس السوداء طالما تريد أن يظل الخادم قيد التشغيل.
- إذا لم تفتح الواجهة تلقائيًا، افتح هذا العنوان يدوياً: http://127.0.0.1:8000/
- إذا كان لديك جدار حماية أو قيود شبكة، تأكد أن `127.0.0.1:8000` مسموح محلياً.
- لتشغيل الواجهة البديلة (Streamlit) — إذا قمت بتثبيت `streamlit` — يمكنك تشغيل:
  & 'D:\AGL\.venv\Scripts\python.exe' -m streamlit run D:\AGL\repo-copy\app_ui.py

دعم وDebug:
- سجل الطلبات يتم تدوينه إلى `logs/requests.log` داخل المجلد الرئيسي للمشروع.
- إن واجهت مشاكل CORS أو عدم استجابة، افتح تيرمنال جديد وشغّل:
  & 'D:\AGL\.venv\Scripts\python.exe' -m http.server 8001
  ثم افتح: http://127.0.0.1:8001/web/index.html

إذا أردت، أستطيع الآن:
- (أ) تشغيل `Start_AGL.bat` هنا لاختبار التشغيل الكامل، أو
- (ب) إزالة أي ملفات مؤقتة إضافية، أو
- (ج) تعديل `Start_AGL.bat` ليشغل الخادم في نافذة منفصلة/خلفية باستخدام `start /B` أو `start /MIN`.
