# AGL Safety & Governance (runtime knobs)

This document summarizes the lightweight runtime safety defaults and helpers
implemented in `infra/safety.py`.

Default env variables (and their defaults):

- AGL_AUTONOMY_ENABLED=1
- AGL_AUTONOMY_BUDGET_SECONDS=1800
- AGL_AUTONOMY_BUDGET_CALLS=200
- AGL_GOAL_PARALLELISM=1
- AGL_ALLOW_NETWORK=0
- AGL_ALLOW_DISK_WRITE=artifacts/,logs/
- AGL_ALLOW_EXEC_SHELL=0
- AGL_SECRETS_PATH=
- AGL_REQUIRE_APPROVAL_FOR={"network":true,"exec":true,"external_write":true}
- AGL_DECISION_NOTES=1
- AGL_RISK_LOG=1
- AGL_SNAPSHOT_ASYNC=1
- AGL_GLOBAL_TIMEOUT_SECONDS=3600

Operational primitives:

- artifacts/controls/STOP: create this file to trigger an immediate kill-switch.
- artifacts/approvals/: the system will write approval requests as JSON here.
  Human operators can edit the JSON to set `"approved": true` to grant an action.
- artifacts/logs/safety.log, decision_notes.jsonl, risk_log.jsonl contain
  the safety logs generated by the runtime.

Usage:

Import `infra.safety` and call helpers, e.g. `infra.safety.allow_network()` or
use `@infra.safety.gated('network')` to protect a function.

The implementation is intentionally minimal and synchronous: approval
requests are created and a human operator is expected to inspect and update
`artifacts/approvals/*.json` to grant actions.

This is a low-risk starting point for governance; you can extend it with
notification hooks, automated approver bots, and stricter auditing later.
