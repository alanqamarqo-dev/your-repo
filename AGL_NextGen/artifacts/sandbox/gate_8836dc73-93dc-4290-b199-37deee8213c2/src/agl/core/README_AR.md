# ⚛️ توثيق النواة (Core Documentation)

**المطور:** حسام هيكل  
**التاريخ:** 2 يناير 2026

---

## 📋 الوصف
مجلد **Core** هو القلب النابض لنظام AGL. يحتوي على المكونات المركزية التي تدير الوعي، التحكم، وتنسيق العمليات بين جميع المحركات الأخرى. لا يتم تنفيذ مهام "فرعية" هنا، بل يتم اتخاذ القرارات الكبرى.

## 📂 محتويات المجلد

### 1. `master_controller.py` (وحدة التحكم الرئيسية)
- **الوظيفة:** نقطة الدخول (Entry Point) للمستخدم.
- **المهام:**
    - عرض القائمة الرئيسية.
    - استقبال أوامر المستخدم.
    - استدعاء `Super Intelligence` لتنفيذ الأوامر.
    - تشغيل الاختبارات التشخيصية.

### 2. `super_intelligence.py` (الذكاء الخارق)
- **الوظيفة:** العقل المدبر.
- **المهام:**
    - تحميل خريطة النظام (System Map).
    - اكتشاف المحركات المتاحة (Auto-Discovery).
    - توجيه الأسئلة إلى المحرك المناسب (Routing).
    - إدارة الوعي الذاتي (Self-Awareness).

### 3. `unified_system.py` (النظام الموحد)
- **الوظيفة:** المجمع (Aggregator).
- **المهام:**
    - يجمع كل القدرات (الذاكرة، الاستدلال، الإبداع) في كائن واحد `UnifiedAGISystem`.
    - يوفر واجهة برمجية موحدة للمطورين لاستخدام النظام.

### 4. `base.py`
- **الوظيفة:** الفئات الأساسية (Base Classes).
- **المهام:** تعريف الهياكل الأساسية التي ترث منها باقي الكائنات.

---

## 🔄 تدفق البيانات في النواة
1. **المستخدم** يرسل أمراً عبر `Master Controller`.
2. **Master Controller** يمرر الأمر إلى `Super Intelligence`.
3. **Super Intelligence** يستشير `Self-Awareness` لمعرفة المحركات المتاحة.
4. يتم إرسال الأمر إلى المحرك المناسب (في مجلد `engines`).
