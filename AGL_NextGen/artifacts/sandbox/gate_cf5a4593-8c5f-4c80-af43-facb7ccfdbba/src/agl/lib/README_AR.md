## 📋 الوصف
مجلد **Lib** يحتوي على البنية التحتية البرمجية التي تعتمد عليها المحركات. هو "صندوق العدة" الذي يوفر وظائف جاهزة للاستخدام، مثل التعامل مع الملفات، الذاكرة، والشبكات.

## 📂 أهم المكتبات

### 1. `unified_lib.py` (المكتبة الموحدة)
- **الوظيفة:** واجهة مركزية لجميع مكتبات Python الخارجية.
- **الميزة:** بدلاً من استيراد `numpy` و `torch` في كل ملف، يتم استدعاؤهم عبر `UnifiedLib.sci.numpy` و `UnifiedLib.ai.torch`. هذا يسهل إدارة الاعتماديات.

### 2. `core_memory/` (الذاكرة المركزية)
- **الوظيفة:** تخزين المعلومات.
- **المكونات:**
    - `Conscious_Bridge.py`: يدير الذاكرة قصيرة المدى (STM) وطويلة المدى (LTM).
    - `memory.db`: قاعدة بيانات SQLite لتخزين الحقائق الدائمة.

### 3. `knowledge_base/` (قاعدة المعرفة)
- **الوظيفة:** أرشيف المعلومات المكتسبة.
- **المكونات:** `Improvement_History.py` (سجل تحسينات النظام لنفسه).

### 4. `utils/` (أدوات مساعدة)
- **الوظيفة:** وظائف صغيرة متكررة.
- **المكونات:** `llm_tools.py` (أدوات بناء روابط LLM).

### 5. `AGL_Paths.py`
- **الوظيفة:** إدارة مسارات النظام.
- **الأهمية:** يضمن أن بايثون يستطيع رؤية مجلد `src` من أي مكان، مما يمنع أخطاء `ModuleNotFoundError`.

---

## 🔄 العلاقة مع باقي النظام
- **المحركات (Engines)** تستدعي **المكتبات (Lib)** لتنفيذ مهامها.
- **النواة (Core)** تستخدم **المكتبات** لتهيئة البيئة وتشغيل النظام.
